# 로그/백테스팅 화면 명세서

## 1. 개요
- **목적**: 매매 및 백테스팅 로그를 분석하고, 데이터를 내보내어 외부에서 활용할 수 있도록 지원하는 화면입니다. 사용자는 매매 전략의 성과를 시각적으로 확인하고 로그를 효율적으로 관리할 수 있습니다.
- **대상 사용자**: 중급 이상의 트레이더 및 데이터 분석가.
- **주요 기능**:
  - 매매 및 백테스팅 로그 상세 조회.
  - 로그 필터링 및 검색.
  - 성과 분석을 위한 시각적 대시보드.
  - 로그 데이터 내보내기(CSV, PDF).

## 2. 기능 요구사항

### 2.1 로그 조회
- **로그 유형**:
  - **매매 로그**: 실제 매매 내역(매수/매도 시간, 가격, 지표 값 등).
  - **백테스팅 로그**: 시뮬레이션 내역(매수/매도 시점, 가상 수익/손실 등).
- **로그 상세 정보**:
  - 로그 유형(매매/백테스팅).
  - 코인(예: BTC/KRW).
  - 전략 이름(예: MFI 보수적).
  - 매수/매도 여부.
  - 거래 시간.
  - 거래 가격.
  - 사용된 지표 값(예: MFI: 18).
  - 수익/손실(백테스팅의 경우).

### 2.2 로그 필터링 및 검색
- **필터링**:
  - 로그 유형(매매/백테스팅).
  - 코인.
  - 전략.
  - 날짜 범위.
- **검색**:
  - 키워드 검색(예: "BTC", "매수").
  - 검색 결과는 실시간으로 리스트에 반영.

### 2.3 성과 분석 대시보드
- **시각적 분석**:
  - 수익률 그래프(일별/주별).
  - MDD(Maximum Drawdown) 표시.
  - 승률 및 평균 수익/손실.
- **대시보드 UI**:
  - 차트: 라인 차트, 바 차트로 시각화.
  - 요약 지표: 총 수익률, MDD, 승률, 거래 횟수.

### 2.4 로그 내보내기
- **내보내기 형식**:
  - **CSV**: 로그 데이터를 표 형식으로 내보냄.
  - **PDF**: 시각적 대시보드와 로그를 포함한 보고서 형식.
- **내보내기 옵션**:
  - 전체 로그 또는 필터링된 로그 선택 가능.
  - 이메일, 클라우드 저장소 등으로 공유 가능.

## 3. UI/UX 요구사항
- **레이아웃**:
  - **상단**: 필터링 및 검색 바.
  - **중앙**: 스크롤 가능한 로그 리스트.
  - **하단**: 성과 분석 대시보드(차트 및 지표).
- **인터랙션**:
  - 로그 항목 탭 시 상세 정보 팝업.
  - 필터링 및 검색은 실시간 반영.
  - 내보내기 버튼은 상단 툴바에 위치.
- **테마**: 다크 모드 기본, 라이트 모드 전환 가능.

## 4. 성능 요구사항
- **로그 로드**: 대량 로그 시 페이징 처리로 빠른 로드.
- **차트 렌더링**: 부드러운 시각화 제공.
- **내보내기**: 백그라운드 처리로 UI 중단 방지.
